# Wraith Desktop 宠物系统设计

日期: 2026-07-18
状态: 已实施

> 展示表面已被 2026-07-19 全局桌宠窗替换（见 `2026-07-19-desktop-pets-global-window-design.md`）：宠物不再是本文档描述的聊天内浮件，而是独立于主窗口的全局桌宠 `BrowserWindow`；本文档描述的来源/导入/校验/注册表/状态映射等后端能力继续有效。

## 目标

为 Wraith Desktop 增加不干扰开发工作流的宠物系统。宠物承担品牌陪伴与 Agent 状态反馈两项职责，支持官方条目、Petdex 兼容包、用户自制包和单张用户图片。

Wraith 默认宠物是 Wraith 自有的轻量机械角色，使用项目自产资源；角色美术细节独立于本规格实现。首个官方可选目录条目为 `Noir Webling`。它不随 Wraith 安装包分发：Wraith 只检测或导入用户已经安装的本地 Petdex 包，以避免将未单独声明授权的宠物资源重新分发。

## 范围

本期交付：

- 设置页的宠物开关、选择、导入、删除和动态风格配置；
- 聊天视图中的紧凑宠物浮件；
- 统一 Agent 状态到宠物状态的映射；
- 单图的非破坏性动态化；
- Wraith 与 Petdex 兼容精灵包的导入和本地发现；
- `Noir Webling` 官方目录项与本地 Petdex 检测。

本期不做：

- 在 Wraith 内自动下载 Petdex 资源、运行 `npx` 或执行第三方代码；
- 使用 AI 生成用户图片的额外姿势；
- 网络宠物市场、账号、上传分享或云同步；
- 重绘或修改 Noir Webling 的美术资源。

## 体验

### 设置页

在既有设置导航新增“宠物”页。页面包含：

- 总开关：关闭后不挂载浮件、不解码宠物图片；
- 宠物库：内置 Wraith 角色、已检测到的 Petdex 条目、用户导入内容；
- 选中宠物的工作中预览；
- 动态开关与单图动态风格（克制、悬浮、活泼、静态）；
- “导入宠物”与删除用户导入宠物操作。

`Noir Webling` 始终可见于官方目录，但仅在检测到有效的 `~/.codex/pets/noir-webling/` 或用户已导入其包时可启用。缺少资源时展示“未安装”说明，不自动下载，不承诺该资源由 Wraith 提供。

### 聊天浮件

宠物默认位于聊天视图右下角、Composer 上方，使用紧凑固定尺寸；不占 transcript 布局空间、不遮挡输入框或审批控件。用户可隐藏、缩放及拖动位置，偏好持久化到桌面端本地设置。

宠物不发送文案、对话气泡、Toast 或错误信息。状态反馈是辅助视觉信号，正式的工具卡、审批面板和错误 banner 保持现有责任。

系统启用“减少动态效果”时，宠物停在静态姿势，仍保留可见性与状态选择功能。

## 宠物来源与注册表

所有宠物进入 renderer 前由统一 `PetRegistry` 标准化为同一条记录：

```text
内置目录 ─┐
Petdex 本地发现 ─┼─> PetRegistry ─> PetPlayer ─> 设置页与聊天浮件
用户导入包 ─────┤
用户导入单图 ───┘
```

来源优先级是用户已导入副本、Petdex 本地目录、官方目录元数据。相同 id 的高优先级来源覆盖低优先级来源。注册表只保存资源路径、来源、能力和元数据；不加载 JavaScript、HTML、SVG 脚本或任意可执行文件。

用户导入后，资源被复制到 Wraith 的应用数据目录，随后不依赖原始文件路径。删除操作只删除 Wraith 管理的导入副本，不修改 `~/.codex/pets` 或任何外部目录。

## 资产格式

### 单图宠物

支持 PNG、JPEG 和 WebP。单图具有 `static` 能力：

- `idle`：轻微呼吸或悬浮；
- `thinking`：缓慢悬浮、微倾和弱强调色光效；
- `tool`：短位移或扫描强调；
- `approval`：静止加低频呼吸；
- `success`：小幅回弹；
- `error`：一次短促晃动后停住。

这些动画仅改变容器的 transform、opacity、filter 和光效，不伪造角色肢体、表情或新的美术帧。动态风格决定幅度与节奏；“静态”关闭所有运动。

### 精灵包

支持 Wraith 包及 Petdex 兼容包：目录或压缩包解包后必须有 `pet.json` 和 `spritesheet.webp` 或 `spritesheet.png`。Petdex 兼容布局为 8 列、9 行、每帧 192×208 px；Wraith manifest 可显式声明布局与状态行，缺失声明时按该兼容布局解释。

状态行使用：`idle`、`wave`、`run`、`failed`、`review`、`jump` 及可选扩展行。`PetPlayer` 把 Wraith 的状态映射到存在的行；行不存在时回退 `idle`，而不拒绝整个包。

## 状态映射

| Wraith 运行状态 | 宠物状态 | 精灵包优先帧 | 单图表现 |
| --- | --- | --- | --- |
| 空闲、无活跃 turn | `idle` | `idle` | 静止或轻呼吸 |
| Thinking / reasoning 流 | `thinking` | `idle` 或扩展 thinking 行 | 缓慢悬浮 |
| 工具执行、浏览、终端任务 | `tool` | `run` | 短扫描或位移 |
| Approval / Plan review 等待用户 | `approval` | `review` | 低频呼吸 |
| turn 成功结束 | `success` | `jump` 或 `wave` | 轻回弹 |
| 失败、提交错误、Agent 异常 | `error` | `failed` | 一次晃动 |

`success` 和 `error` 是短暂状态，结束后回到 `idle`；一个新 turn 开始时立即覆盖旧状态。宠物不暴露 prompt、工具参数、文件路径、错误详情或模型内容。

## 导入、安全与容错

- 仅接受受支持的图片 MIME 与文件签名；拒绝 SVG、HTML、脚本和未知二进制；
- 单图最大 8 MiB、精灵图最大 16 MiB，任一图像边长不超过 4096 px，解压后的包最大 24 MiB 且至多 64 个文件；超限即拒绝，防止图片炸弹和 Zip Slip；
- 归档解压路径必须留在临时导入目录内；
- manifest 的 id、显示名和相对资源路径使用严格 schema 校验；
- 新资源完成验证和复制后才替换注册表记录，失败时保留原来的可用宠物；
- 解码或播放任一帧失败时退回已有缩略图或静态 `idle`，并在设置页给出可操作错误；
- Petdex 目录不可读、被删除或更新时，记录变为未安装，不影响其他宠物或当前会话。

## 持久化

设置保存：是否启用、选中宠物 id、位置、缩放、单图动态风格、是否尊重减少动态效果。每个宠物记录保存来源、导入时间、能力、相对资源目录和校验结果；不保存图像外的用户内容。

## 测试与验收

单元测试：

- manifest/schema、路径和来源优先级；
- Petdex 8×9 帧索引与缺帧回退；
- Wraith 状态 reducer 与短暂 success/error 的复位；
- 单图动态风格到 motion profile 的映射；
- 图片/归档拒绝规则和导入原子性。

组件与集成测试：

- 设置页开关、选择、导入失败、删除和偏好持久化；
- 发现 `~/.codex/pets/noir-webling` 后官方目录条目可用；
- 宠物浮件不改变消息区与 Composer 的尺寸；
- 减少动态效果时无 CSS/JS 动画运行。

手工验收：

- 浅色、深色、窄窗口、侧栏展开/收起；
- 聊天、工具面板、审批和 Plan review 期间无重叠；
- Petdex Noir Webling 使用原始精灵图显示，不被 Wraith 重绘；
- 单图导入在无透明背景、透明背景和大尺寸图片下均得到明确结果；
- 关闭宠物后刷新应用，浮件不出现且不加载资源。

## 风险与决策

- Petdex 仓库代码是 MIT，但宠物资源按提交者各自许可。故 Wraith 兼容其格式和扫描用户本地安装目录，不将 Noir Webling 资源作为 Wraith 发布物内置。
- 首期不运行 `npx`，避免下载、子进程、遥测和供应链行为进入桌面 App。
- 单图动画故意限于容器动效；真实逐帧姿势属于用户提供精灵包或未来获得明确授权的生成工作流。
