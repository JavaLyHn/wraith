# 工具面板宽屏布局(内容居中定宽)设计稿

日期:2026-07-12
状态:已与用户确认设计(方向 + 宽度),待写实现计划
所属:桌面布局增强;由「侧边栏折叠」特性暴露 —— 折叠后主内容变很宽,工具面板内容仍靠左、右侧大片空白。

## 目标

让工具面板(设置/Provider/插件 等)在宽视口(折叠侧栏后)下不再内容靠左显空:给每个面板的**内容区**一个舒适 `max-width` 并水平居中,**头部(← 返回对话 条)保持全宽**。窄视口(侧栏展开)下与现状基本一致(内容未达上限,自然铺满)。

## 确认的决策(用户)

1. 方向:**内容居中 + 舒适定宽**(非撑满全宽,非按面板类型分别处理)。
2. 宽度:**max-width 1040px**。
3. 头部(`← 返回对话` / 标题条)保持**全宽**,不随内容居中。

## 范围

- **做**:`tokens.css` 加共享类 `.panel-content`;给 12 个工具面板(plugins/automations/im-gateway/providers/skills/memory/snapshots/tasks/policy/browser/rag/settings 对应的 `*Panel.tsx`)的「头部下方内容区」套上该类使其居中定宽。
- **不做**(YAGNI):按面板类型分别定宽、多列重排、改任何面板功能/数据流、动聊天视图、动侧栏/RightDock/终端。

## 设计

### 共享类(单一真源)
`desktop/src/renderer/styles/tokens.css` 追加:
```css
/* 工具面板内容区:宽视口下舒适定宽居中,头部仍全宽。改宽度只改此处。 */
.panel-content { width: 100%; max-width: 1040px; margin-inline: auto; }
```
选用共享类(而非各处写 `max-w-[1040px] mx-auto`)以便一处调宽、风格统一。

### 面板改法(统一模式)
每个 `*Panel.tsx` 结构均为 `flex flex-col`:全宽头部 + body。改动只在 **body/内容区** 加 `panel-content` 居中,不碰头部:
- **单栏内容面板**(多数):内容区通常是 `overflow-y-auto` 的滚动容器。在其**内部**包一层 `<div className="panel-content …">`(或给已有的内容 wrapper 加 `panel-content`),使内容居中、滚动条仍在窗口右缘自然位置。若面板无内层 wrapper,则新增一层承载原内容。
- **Settings**(`SettingsPanel.tsx`,两栏 subnav+content):把「subnav + content」这**整个两栏 body**(`<div className="flex min-h-0 flex-1">`)作为一个块居中定宽 —— 给它加 `panel-content`(即 `mx-auto max-w-[1040px] w-full`),两栏整体居中,两侧留白;subnav 的 `border-r` 落在居中块内部,观感为一张居中的设置卡。

具体每个面板套在哪一层由实现计划逐个对准(各 body 结构略异);统一原则:**头部全宽、内容区 `panel-content` 居中、不改功能**。

## 错误 / 边界
- 窄视口(侧栏展开、窗口不宽):内容宽度小于 1040 时 `panel-content` 自然铺满,视觉与现状一致,无回退。
- 滚动:优先让内容(而非滚动容器本身)居中,使滚动条保持在窗口右缘;若某面板结构上更简单是给滚动容器加类,也可接受(滚动条随居中块),逐面板取顺手且不破坏滚动的那种。
- 头部/横幅(如错误条、统计条)保持全宽,不居中。
- 聊天视图、侧栏、RightDock、终端:不受影响。

## 测试
- 纯布局改动,无单元测试;`npm run typecheck` exit 0 + 既有 `npm test`(646)保绿。
- 眼验:折叠侧栏(宽视口)下逐个进 12 个面板,内容居中、两侧留白均衡、不再靠左空;展开侧栏(窄视口)下内容自然铺满、与现状一致;深浅主题正常;头部仍全宽。

## 交付后 / 风险
- 12 个面板逐个套类,body 结构差异导致「套在哪一层」需逐个对准(尤其两栏 Settings、带横幅/统计条的面板);逐面板核对滚动与头部不受影响。
- 宽度 1040 为单一真源常量,后续易调。
